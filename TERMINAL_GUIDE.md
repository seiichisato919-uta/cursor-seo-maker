# ターミナルの使い方 - 超わかりやすいガイド

## ステップ1: ターミナルを開く

### 方法1: Spotlight検索を使う（一番簡単）

1. キーボードの `Command + Space`（コマンドキーとスペースキー）を同時に押す
2. 「ターミナル」または「Terminal」と入力
3. 「ターミナル」アプリが表示されたら、Enterキーを押す
4. ターミナルウィンドウが開きます

### 方法2: アプリケーションから開く

1. Finderを開く
2. 左側のサイドバーから「アプリケーション」をクリック
3. 「ユーティリティ」フォルダを開く
4. 「ターミナル」アプリをダブルクリック

### 方法3: VS CodeやCursorから開く

1. VS CodeやCursorを開く
2. メニューから「ターミナル」→「新しいターミナル」を選択
   - または、ショートカット：`Ctrl + Shift + ``（バッククォート）`
   - または、`Ctrl + J`（Macの場合）

## ステップ2: プロジェクトフォルダに移動

ターミナルが開いたら、プロジェクトフォルダに移動します。

### コマンドを入力

ターミナルに以下のコマンドを入力して、Enterキーを押します：

```bash
cd "/Users/katanoyousuke/Cursorテスト/新しいフォルダ/汎用型SEO記事作成システム"
```

**入力のコツ**:
- `cd` の後にスペースを1つ入れる
- パス全体をダブルクォート `"` で囲む
- パスをコピー&ペーストすると簡単です

### 確認

正しく移動できたか確認するには、以下のコマンドを入力：

```bash
pwd
```

Enterキーを押すと、現在のフォルダのパスが表示されます。

## ステップ3: 開発サーバーが実行中か確認

### 確認方法

ターミナルを見て、以下のような表示があるか確認：

```
> npm run dev

> seo-article-creator@1.0.0 dev
> next dev

  ▲ Next.js 14.2.0
  - Local:        http://localhost:3000
  - Ready in 2.3s
```

このような表示があれば、開発サーバーが実行中です。

### 実行中の場合：停止する

1. ターミナルウィンドウをクリックして選択
2. `Ctrl + C`（コントロールキーを押しながらCキー）を押す
3. サーバーが停止します

### 実行中でない場合：そのまま次へ

何も表示されていない、または `$` や `%` の記号だけが表示されている場合は、サーバーは実行されていません。そのまま次へ進みます。

## ステップ4: 開発サーバーを起動

ターミナルに以下のコマンドを入力して、Enterキーを押します：

```bash
npm run dev
```

### 実行結果

正常に起動すると、以下のような表示が出ます：

```
> npm run dev

> seo-article-creator@1.0.0 dev
> next dev

  ▲ Next.js 14.2.0
  - Local:        http://localhost:3000
  - Ready in 2.3s
```

### エラーが出た場合

もしエラーが出た場合は、エラーメッセージを確認してください。

よくあるエラー：
- `command not found: npm` → Node.jsがインストールされていません
- `Cannot find module` → 依存関係がインストールされていません（`npm install` を実行）

## ステップ5: ブラウザで確認

1. ブラウザを開く
2. アドレスバーに `http://localhost:3000` と入力
3. Enterキーを押す
4. アプリケーションが表示されれば成功です！

## ターミナルの基本操作

### コマンドの入力方法

1. ターミナルウィンドウをクリックして選択
2. コマンドを入力
3. Enterキーを押して実行

### よく使うコマンド

| コマンド | 説明 |
|---------|------|
| `pwd` | 現在のフォルダのパスを表示 |
| `ls` | 現在のフォルダのファイル一覧を表示 |
| `cd フォルダ名` | フォルダに移動 |
| `cd ..` | 一つ上のフォルダに移動 |
| `npm run dev` | 開発サーバーを起動 |
| `Ctrl + C` | 実行中のコマンドを停止 |

### コマンドの履歴

- 上矢印キー（↑）：前に入力したコマンドを表示
- 下矢印キー（↓）：次のコマンドを表示

## トラブルシューティング

### Q: ターミナルが開けない
A: Spotlight検索（`Command + Space`）で「ターミナル」と検索してください。

### Q: コマンドが実行されない
A: Enterキーを押したか確認してください。コマンドを入力しただけでは実行されません。

### Q: 「command not found」と表示される
A: コマンドのスペルが正しいか確認してください。または、Node.jsがインストールされているか確認してください。

### Q: 開発サーバーが起動しない
A: 
1. プロジェクトフォルダに正しく移動しているか確認（`pwd` コマンドで確認）
2. `npm install` を実行して依存関係をインストール
3. `.env.local` ファイルが正しく設定されているか確認

### Q: ターミナルを閉じても大丈夫？
A: ターミナルを閉じると、開発サーバーも停止します。再度起動する場合は、`npm run dev` を実行してください。
